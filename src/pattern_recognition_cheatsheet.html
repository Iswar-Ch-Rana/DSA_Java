Step 2: Analyze Input Format

Tree/Binary Tree/BST

  • DFS for: all paths, recursive exploration, preorder/inorder/postorder
  • BFS for: level-by-level, shortest path in unweighted tree
  • Consider tree properties and parent-child relationships

Graph (nodes + edges)

  • BFS for shortest path
  • DFS for connected components
  • Union Find for "connected components" or "number of groups"
  • Topological Sort for dependencies
  • Dijkstra/Bellman-Ford for weighted shortest path

2D Grid/Matrix

  • DFS/BFS for "islands" problems
  • Union Find for connected regions
  • Dynamic Programming for path problems
  • Consider 4-directional or 8-directional movement

Sorted Array

  • Two Pointers technique
  • Binary Search
  • Greedy approach
  • Merge operations

String

  • Two Pointers for palindromes
  • Sliding Window for substrings
  • Trie for word problems
  • Stack for parentheses/brackets
  • KMP/Z-algorithm for pattern matching

Linked List

  • Two Pointers (fast/slow)
  • Dummy Node techniques
  • Cycle Detection (Floyd's algorithm)
  • Reversal techniques

Step 3: Analyze Output Format

List of Lists (combinations, subsets, paths)

‚Üí Backtracking is almost always the answer. Generate all possibilities using recursion with choice/no-choice pattern.

Single Number (max/min profit, cost, ways, jumps)

‚Üí Dynamic Programming for optimization, Greedy for local optimal choices, Mathematical approach for counting.

Modified Array/String (in-place operations)

‚Üí Two Pointers for in-place modifications, space optimization techniques.

Ordered List (sorted sequence, valid task order)

‚Üí Sorting with custom comparators, Topological Sort for dependencies, Heap for maintaining order.

Boolean (true/false, possible/impossible)

‚Üí Dynamic Programming with boolean states, Greedy validation, or Graph reachability.
